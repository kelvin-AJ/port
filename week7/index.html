Notes from reading exercises and links to practice.

  • Further Functions(Examples in JS file) :
    1. The call() method can be used to set the value of 'this' inside a function to an object that is provided as the first argument.
    2. If a function doesn't refer to an object as this in its body, it can still be called using the call() method, but you need provide null as its first argument;
    3. The 'apply()' method works the same way, only that the parameters are passed in an array
    4. We can add preperties to functions just as we can to normal objects
    5. Memoization involves the use of a "cache" property assigned to a function to store results whenever the function runs. This is especially useful for functions that take time to run so it saves time by simply returning values already stored for reused arguments.
    6. IIFE are anonymous functions that are called immediately they are defined
    7. Placing any code that uses any temporary variables inside an IIFE will ensure it's only available while the IIFE is invoked, then it will disappear. This reduces the chances of the global namespace being polluted.
    8. IIFEs are alse useful to set up initialization code that won't be needed again.
    9. To avoid enforcing strict mode on every javascript in the file, the recommended way to use strict mode is to place all strict code inside an IIFE.
    10. functions in javascript can, call, define and even redefine themselves.
    11. With Init-Time Branching, we can redefine a function based on the presence of a feature.
    12. A recursive function is one that calls itself until a certain condition is met.
    13. callback functions are functions passed to other functions as arguments and then invoked inside the function they are passed to
    14. Call back hell occurs when we have to nest multiple callback functions in each other, usually because of their asynchronous nature.
    15. Event-driven Asynchronous Programming ensure that waiting for tasks such as events and other time taking functions does not hold up execution of other parts of the program.
    16. Promises represent the future result od asynchronous operations. Like a placeholder while we wait for the outcomes of an asynchronous task. A settled promise can result in two outcomes: Resolved or Rejected.
    17. We can chain multiple promises.
    18. Async functions were added to the ES2017 specification. These functions are preceded by the async keyword and allow you to write asynchronous code as if it was synchronous.
    19. We can have functions return other functions
    20. A closure gives a function access to all the variables of its parent function or "it's birthplace" even after it has been reassigned.
    21. Pure functions adhere to the following rules:
      • The return value of a pure function should only depend on the values provided as arguments. It doesn't rely on values from somewhere else in the program.
      • There are no side-effects. A pure function doesn't change any values or data elsewhere in the program. It only makes non-destructive data transformations and returns new values, rather than altering any of the underlying data.
      • Referential transparency. Given the same arguments, a pure function will always return the same result.
    22. Higher-Order Functions are functions that accept other functions as arguments and/or returns another function.
    23. Curring involves nesting functions into other functions such that the inititial function can take one parameter and then return a function that takes another and the chain can go on until all parameter are available at the last function.This "function call chaining" is useful when other needed parameter are ot readily available.
