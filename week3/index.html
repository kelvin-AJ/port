  • More On Objects
    1. An object in JavaScript is a self-contained set of related values. They act as a collection of named properties that map to any JavaScript value(data type: string, boolean, array, objects, integers and so on).
    2. We acn create objects by using a pair of curly braces or using the Object() constructor
    3. We lookup those values by using those names(keys). We can either use the dot notation : ObjectName.key or square brackets ObjectName[key]. We can use the square brackets for computed properties.
    4. Object keys follow JavaScript Naming Conventions.
    5. using the in operator, we can check whether an object has a particular property. We can also use the .hasOwnProperty('property')
    6. We can use a for in loop to loop over keys of an object. Object.keys(), Object.values(), and Object.entries() return an array of the keys, values and entries of the Object passed in as an argument respectively. If we destructure the entries into [keys, values] while looping through them, we get acces to the keys and values.
    7. We can add or change properties by simply assigning or reassigning those keys to the new values.
    8. Objects are copied by reference.
    9. We can desstructure objects as funuction parameters so the order of passing those parameter do not matter. It is of great advantage whne we're working with functions that require alor of parameters
    10. Defining functions or variables generally inside Objects made it very unlikely to have nameing collision issues. When this function are to be used, they have to be preceeded by the namespace(object name).

  • Events
    1. We handle this events when they occur with event handlers. For example, we can alert a message to the user when a button is clicked. The "click" is the event in this situation.
    2. We can use inline event handlers like the 'onclick' which we insert as an attribute directly in the HTML tags and assign them to the JavaScript we want to run. Although, inline event handlers are best avoided.
    3. Using EventListeners are a recommended way of dealing with events mainly because they allow multiple functions to be attached to an event.
    4. The addEventListener method takes in two arguments. The first is the event that we want to listento. The other is the function be executed when the event occurs .addEventListener('event', function );
    5. (practice in script)
    6. There are no swipe events. These need to be created by using a combination of touchstart, touchmove, and touchleave events that monitor the distance and direction moved from start to finish of a touch event.
    7. We can remove event listeners by using the removeEventListener() method
      8. but we can't remove anonymous functiosn if we plan to remove them.
    8. We can prevent deafult beahaviors to stop the page from doing what it usually does for links or form submission by using the preventDefault()
    9. There are two phases of event propagation. Capturing and Bubbling. When a button is clicked for example, the event goes through a capturing phase first(the click evene happens on all parent element from the top down to the target) the bubbling phase is basically the reverse.
    10. Event delegation helps attach event listeners to parent elements in order to capture events that are triggered by the child element.
