  • CSS3 Transforms and Transitions
    1. The CSS3 transform property lets us translate, scale and/or skew any element on the webpage.
    2. transform: translate(-10px, -10px) :
      hover to translate
      To move elements vertically or horizontally, we use the translateX or translateY property. Transform don't work on inline elements
    3. transform: scale(0.8, 1.5) :
      hover to scale
      We can alse use scaleX and scaleY to scale horizontally or vertically.
    4. Declaring a second transform to the same element would overwrite the first. To add multiple transform attributes, separate attibutes with a space.
    5. Transforming does not cause a reflow
    6. transform: rotate(30deg) :
      hover to rotate
      rotation happens clockwise.
    7. transform: skew(-30deg, -10deg) :
      hover to skew
    8. we can manipulate the origin of our transform using the tranform-origin property. this is especially useful for shapes like cicles or ellipses that look the same if their center of roation is at their center.
    9. The order fo transform functions matter, The effect is applied one after the other. Almost like one function returns a new version of that element that has been transformed
    10. CSS Transitions allow the values of CSS properties to change over time. The transition-property property defines the CSS properties of the element that should be transitioned. The transition-duration property sets how long the transition will take. The transition-timing-function lets you control the pace of the transition in more detail.
    11. We cal have mulitple transition propertues by separating them with commas.
    12. Generally, it is best to use CSS for simple-state changes in a mobile environment. But it's still better to employ JavaScript for intricate, stateful UIs, and when you do, you'll likely want to use a JavaScript animation library to help with manageability and performance.
    13. Animated element using keyframes:
    14. The animation-fill-mode property defines what happens before the first animation iteration begins and after the last animation iteration concludes
    15. When using the shorthand property, animation names that are also animation property key term such as forwards, running, or alternate should not be used.
