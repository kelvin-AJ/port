  • Forms
    1. The span attribute is the URL where the form data would be submitted
    2. We can access form element by using document.forms which returns an HTML collection of all forms in the document. we can also use document.getElementsByTagname("form")[0]
    3. we can use the name attribute to identify a form. We can do this with both dot "." and bracket "[]" notation. Using the bracket notation is more advised to avoid selection form properties.
    4. the elements method returns an HTML collection of all elements contained in the form.
    5. We can also access the form controls using their name attributes as if it was a property of the form object. so if set the name attribute of a first Name input to firstName, we can access it like form.firstName
    6. form.submit() would submit the form automatically. A form can be submitted manually by using an actual button or an input type of submit.
    7. form.reset() resets all form controls back to their initial values. a button type with the attribute of reset does the same thing. Reset buttons are generally NOT adviced. You don't want the users to wipe out all the data they entered when it's not yet saved.
    8. form.action sets the action attribute of a form. (form events in console)
  • Object Oriented Programming (OOP)
    1. Object-oriented programming is often used to model representations of objects in the real world.
    2. Encapsulation involves keeping all programming logic hidden(encapsulated) in an object and then providing methods to the users without them having to understand how it works.
    3. Polymorphism: various objects can share the same methods and can override those methods based on what they do. How the on button for a light bulb turns the light on and another on the button for a fan would get it to spin. "same method, different implementation".
    4. Inheritance provides us be the ability to take an object that already exists then inherit all its methods and properties as if they were defined by the new object, then add methods and properties to it.
    5. Classes are like a blueprint for creating objects.
    6. The constructor function returns the object we want to create.
    7. We create instances of constructor functions with the following syntax
    8. this keyword represents the object that is returned.
    9. ES6 class declaration syntax does the same thing as a constructor function but looks similar to classes in class-based programming languages
    10. all code in a class definition is implicitly in strict mode, so doesn't need the 'use strict' statement at the start of the file.
    11. This means that every class has a prototype property that is shared by every instance of the class. So any properties or methods of a class’s prototype can be accessed by every object instantiated by that class.
    12. It is impossible to overwrite the prototype by assigning it to a new object literal if class declarations are used. We can overwrite prototypes if constructor functions are used. Because of the danger that comes with reassigning prototypes, class declarations are recommended.
    13. Properties can be reassigned.
    14. When a property or a method is called, The engine would check if that object has that method. If it doesn't, it will look it up by 'going up the prototype chain until it finds it.
    15. By default, JavaScript object methods are public. They can be changed externally simply by reassigning. We use getters and setters to access and modify private properties.
    16. enumerable object properties show up when a for-in loop is used to loop through that object’s properties.
    17. A class can inherit from another class using the 'extends' keyword in a class declaration.
    18. When a method is called on a primitive value, JavaScript creates a wrapper object for the primitive, which converts it into an object and then calls the method on the object.
    19. monkey-patching involves adding our own methods to the prototype of inbuilt objects. It should be avoided because it can cause unexpected behavior in javascript. A way to avoid monkey-patching is by using 'extends' so you inherit all methods of that inbuilt objects.
    20. Instead of using assignment, we can add properties to an object using the Object.defineProperty() method so we can explicitly set it's property descriptor
    21. A mixin is a way of adding properties and methods of some objects to another object without using inheritance.
    22. object methods returning 'this' make it easy to chain methods.
    23. We can borrow methods by using the call method. This lets us assign the 'this' keyword to the object we pass in as the parameter.
  • Modular JavaScript
    1. that provides functions and methods that can then be used in other files and by other modules
    2. "Keeping code modular helps to make it more loosely coupled and interchangeable, meaning you can easily swap one module for another without affecting other parts of a project."
    3. All code in modules is always in strict mode.
    4. A module has its own global scope, so any variables created in the top-level of a module can only be accessed within that module.
    5. The value of this in the top level of a module is undefined.
    6. Default exports are single variable, function or class in a module that can be imported without having to be explicitly named.
    7. Having more than one default export will result in a syntax error.
    8. Node.js uses Common Js modules.
